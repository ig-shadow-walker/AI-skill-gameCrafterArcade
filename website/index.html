GameCrafter Arcade

Design and build arcade-style 2D and 3D games—score, waves, short sessions, and tight restart loops—for the web, tablets, mobile, and desktop. The default stack is Three.js with TypeScript and Vite when you implement in this repo.

This project ships the arcade-gamecrafter skill for Cursor and Claude Code: a structured pipeline from idea → design → systems → build spec, tuned for reflex and “one more run” games—not sprawling RPG scope unless you scope an arcade slice.

Workflow (top-down)

Agents following the skill produce concrete artifacts at each step—and check in on vibe and lighting while implementing.

  1. Idea — Pitch, platforms, session length, inputs.
  2. Design — Core loop, camera/controls, fail state, pacing.
  3. Systems — World, player, combat, enemies, run flow, audio.
  4. Content — Geometry, layout, copy, placeholders.
  5. Presentation — HUD, juice, accessibility.
  6. Scope — MVP vs stretch.
  7. Build spec — Layout, loop, config, tests—then implement MVP first.

For shooting games, the skill expects visible feedback on each shot (e.g. muzzle flash) unless you opt out.

Claude Code

  3. For long templates, point the agent at reference.md.

Template roadmap (A–H)

When scaffolding the repo itself, agents follow roadmap.md in order. In short:

  • A–B — Vite + TypeScript + Three.js, resize/DPR, timestep.
  • C–D — Input abstraction, HUD, src/game layout, example build spec.
  • E — Minimal arcade reference game.
  • F–H — PWA, optional wrapper, skill/code sync, CI.
